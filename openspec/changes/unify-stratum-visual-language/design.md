## 背景

Stratum Web 当前包含两个产品界面：概览路由与隆中对聊天。两者共享导航和全局 token，但概览页呈现为柔和的多色品牌页面，聊天页则增加了胶囊形、带阴影的工具栏以及另一套面板处理方式。现有 CSS 还保留了多色径向色块、固定颗粒叠层、深阴影和混杂的圆角职责，与项目文档中“克制的工程工作台”方向相冲突。

本次变更将 Lovable 作为近期视觉基础。Lovable 提供产品颜色角色、基于透明度的中性色层级、边框主导的纵深、排版参数和圆角规范。现有 Noto Sans 与 Nunito Sans 字体资源继续使用。Stratum 多色配色仅保留给品牌标识和后续有明确目的的特效，不用于当前产品表面或控件。

实现必须保留现有路由结构、固定导航与输入区行为、隆中对居中单列、浮层历史记录、文档滚动、渐进式透明、审批事实、本地化和 API 驱动的对话状态。项目策略要求保护可复用的第三方组件内部实现。

## 目标与非目标

**目标：**

- 通过共享的语义配色、排版、表面、形状和动效规则，让概览页与隆中对呈现为一个产品。
- 直接采用 Lovable 浅色配色，并从相同的奶油色与炭黑色角色派生一致的暗色配色。
- 保留当前字体文件，同时采用 Lovable 字阶、字重、行高、字距和响应式展示字号。
- 使用浅层、边框主导的容纳方式和有明确文档的圆角系统替代装饰性纵深。
- 保证语义状态色可访问，并与品牌色或装饰色分离。
- 保留当前产品行为，通过 token 和使用方适配受保护组件。
- 将下载的 Lovable DESIGN.md 保存为参考，同时保持 `stratum-web/DESIGN.md` 的权威地位。

**非目标：**

- 使用 Camera Plain 或其他新字体替换 Noto Sans 或 Nunito Sans。
- 添加新的视觉特效、环境动效、渐变装饰或滚动编排。
- 重做路由信息架构、导航标签、后端协议或对话行为。
- 添加永久聊天侧栏、事件列、伪造的运行数据或静态假产品预览。
- 未经单独批准直接修改 `app/components/ui`、`app/components/react-bits` 或 `app/components/ai-elements` 内部实现。

## 决策

### 使用 Lovable 作为产品配色层

浅色主题将语义 token 映射到以下 Lovable 角色：

- 画布与默认表面：奶油色 `#f7f4ed`
- 主文本与主要操作：炭黑色 `#1c1c1c`
- 操作文本与高对比强调：米白色 `#fcfbf8`
- 弱化文本：柔和灰色 `#5f5f5d`
- 静态边框与分隔线：浅奶油色 `#eceae4`
- 交互边框：`rgba(28, 28, 28, 0.4)`
- 键盘焦点：`rgba(59, 130, 246, 0.5)`

次级色调角色将通过炭黑色透明度派生，而不是使用互不相关的灰色色相。暗色主题反转相同的奶油色与炭黑色关系：炭黑色成为画布，米白色或奶油色成为主文本，奶油色透明度用于构建表面、弱化文本和边框。暗色主题不再保留当前的蓝灰色和紫灰色产品表面。

成功、警告和危险等语义色继续保持独立，因为它们用于传达产品状态。这些颜色必须满足对比度要求，且不能成为装饰性强调色。

备选方案：继续使用 Baltic Blue 作为主要操作色。未采用，因为用户选择在第一阶段全面使用 Lovable 产品配色。Baltic Blue 与现有多色 token 转移到保留的特效层。

### 分离产品色、语义色与特效色命名空间

Token 将被组织为三个概念层：

1. 产品 token：将 Lovable 颜色映射到画布、表面、文本、边框、焦点和操作角色。
2. 语义 token：表示成功、警告、危险和信息状态。
3. 特效 token：保留 Stratum 现有多色配色，用于品牌标识和未来按需启用的视觉特效。

本次变更中，特效 token 不得用于页面背景、导航、输入区表面、普通卡片、正文或主要控件。现有全局多色径向背景和固定颗粒叠层将从产品基础层移除。Stratum 标识可以继续保留现有多色处理。

备选方案：删除旧配色。未采用，因为它仍是后续有目的特效可以使用的品牌资产。

### 采用 Lovable 排版参数但不更换字体资源

排版角色使用以下数值：

| 角色 | 字号 | 字重 | 行高 | 字距 |
| --- | --- | --- | --- | --- |
| Hero 展示标题 | 桌面端 60px，平板端 48px，移动端 36px | 600 | 1.00-1.10 | 桌面端 -1.5px，按比例缩放 |
| 章节标题 | 48px | 600 | 1.00 | -1.2px |
| 次级标题 | 36px | 600 | 1.10 | -0.9px |
| 卡片标题 | 20px | 400 | 1.25 | 默认 |
| 大号正文 | 18px | 400 | 1.38 | 默认 |
| 正文与标准操作 | 16px | 400 | 1.50 | 默认 |
| 说明文字与紧凑控件 | 14px | 400 | 1.50 | 默认 |

正常层级只使用 400 与 600 两档字重。Noto Sans 继续用于展示文本和中日韩文字，Nunito Sans 继续用于拉丁正文与界面文本。共享语义工具类用于避免不同路由自行定义局部字号。

备选方案：引入 Camera Plain 以完全复刻 Lovable。未采用，因为用户要求的是 Lovable 字距与字号，而不是替换字体家族；新字体还会引入授权、加载和多语言一致性问题。

### 使用浅层纵深与明确的形状系统

产品容纳方式遵循 Lovable 的浅层模型：

- 普通内容组保持平面，通过间距组织层级。
- 控件和紧凑菜单使用 6px 圆角。
- 标准卡片、输入区表面和图像容器使用 12px 圆角。
- 大型浮层容器可以使用 16px 圆角。
- 全圆角仅用于圆形图标操作、开关，或形状能够表达行为的控件。
- 静态容器使用 `#eceae4` 或对应暗色值，不使用投影。
- 浮层可以使用克制的焦点或纵深阴影。

两个路由的导航使用相同的表面、边框、排版和圆角处理。其响应式宽度仍可为满足隆中对居中单列约束而变化，但材质语言不能在概览页的透明导航与聊天页的阴影胶囊之间切换。

备选方案：保留聊天胶囊作为标志性组件。未采用，因为它是路由间视觉割裂最明显的来源，也与现有导航设计要求冲突。

### 围绕产品状态统一动效

Stratum 自有界面的普通悬停、焦点、开合、选择和路由反馈目标时长为 150-250ms。已有编排需求的 Stratum 自有动效继续优先使用 GSAP。简单状态反馈使用 CSS 过渡即可。受保护第三方组件内部现有的 Motion 用法不在本次变更范围内。

动效应尽量只使用 transform 与 opacity，并为减少动态效果提供立即或近乎立即的最终状态。路由切换必须保留方向感，但不展示冗长的页面入场序列。本次变更不增加环境式或永久循环特效。

备选方案：保留当前所有 350-550ms 动画序列。未采用，因为产品定位强调任务流和简短的状态反馈。

### 通过 token 与 Stratum 自有使用方完成适配

全局语义 token 与 Stratum 自有组件负责视觉迁移。受保护的可复用组件内部保持不变。当受保护组件的默认样式不适用时，其 Stratum 自有调用方通过 props、CSS 变量、限定在使用方的选择器或包装层完成适配。

一次性命令 `npx getdesign@latest add lovable` 将在 `stratum-web` 中运行。由于权威的 `stratum-web/DESIGN.md` 已存在，生成的 `stratum-web/lovable/DESIGN.md` 将作为参考保留，而不是替换项目规则。权威设计文档将汇总已采纳的 Lovable 规则与 Stratum 特有的产品约束。

备选方案：使用生成模板覆盖 `stratum-web/DESIGN.md`。未采用，因为这会丢失产品特有的布局、行为、可访问性和运行数据约束。

## 风险与权衡

- [暖奶油色可能让技术产品显得偏编辑化] -> 一致使用配色，同时保持产品密度、清晰边框、克制圆角和具体的界面文案。
- [Lovable 参考以浅色为主，没有完整规定暗色产品表面] -> 从相同的奶油色和炭黑色角色派生暗色 token，目视验证两种主题，并记录最终语义值。
- [全局排版调整可能导致中文导航和紧凑控件溢出] -> 在桌面端与移动端验证中英文，只在现有行为允许时截断文本，并保证控件满足触控尺寸要求。
- [使用方覆盖可能逐渐难以维护] -> 优先使用语义 CSS 变量和小型 Stratum 自有包装层；仅在受保护组件没有合适 API 时使用较长的任意选择器链。
- [移除装饰性背景后，第一阶段的品牌感可能减弱] -> 将干净的 Lovable 基础视为首个里程碑；仅在后续独立变更中，围绕明确的沟通目的添加 Stratum 特效。
- [同时修改明暗主题会扩大视觉回归范围] -> 为两个路由、两种主题和移动端布局保存前后截图，并按 token 优先的阶段逐步迁移。

## 迁移计划

1. 按项目策略创建独立实现 worktree 和非 main 分支。
2. 在 `stratum-web` 中运行 getdesign 命令，并将生成的 Lovable 文档保留为参考。
3. 更新 `stratum-web/DESIGN.md`，记录权威的三层配色模型、排版参数、形状职责和动效限制。
4. 首先替换全局语义 token 和基础背景，包括浅色与暗色值。
5. 依次迁移共享导航与路由过渡、概览路由、隆中对输入区和历史表面。
6. 验证响应式行为、中英文、明暗主题、焦点对比度、减少动态效果、类型检查和生产构建。
7. 回滚时还原 token 与 Stratum 自有组件变更；参考 DESIGN.md 不影响运行时，可以继续保留。

## 待确认问题

无。已确定的方向是采用 Lovable 产品配色和排版参数、保留当前字体资源，并将 Stratum 配色留给后续特效。
